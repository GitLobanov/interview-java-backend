#### Для чего используется сервер приложений Tomcat?

Apache Tomcat — это сервер приложений, который предназначен для выполнения Java-сервлетов и JavaServer Pages (JSP). Он обеспечивает контейнер для сервлетов, который управляет их жизненным циклом, а также отвечает за обработку HTTP-запросов и отправку HTTP-ответов. Tomcat используется для размещения Java-веб-приложений и предоставляет средства для работы с динамическими веб-страницами.
#### Как осуществляется деплой WAR-файла на сервер Tomcat?

- Для деплоя WAR-файла в Tomcat, нужно выполнить следующие шаги:
    1. Поместите WAR-файл в каталог `webapps` сервера Tomcat.
    2. При запуске Tomcat автоматически развернет WAR-файл в подкаталог с таким же именем, как у файла (например, для `myapp.war` будет создана папка `myapp`).
    3. После развертывания сервер будет доступен по адресу: `http://localhost:8080/myapp/`.
- Также можно развернуть приложение с помощью админ-панели Tomcat, если она активирована.
#### Что такое сервлет в контексте Java EE?

Сервлет — это класс Java, который обрабатывает HTTP-запросы в веб-приложениях. Сервлеты расширяют функциональность сервера веб-приложений, принимая запросы от клиента, выполняя логику обработки и возвращая результаты в виде HTTP-ответа. Сервлеты являются основой динамических веб-приложений в Java EE, обеспечивая взаимодействие между веб-страницами и сервером.
#### Что такое файл server.xml в Tomcat и для чего он используется?

Файл `server.xml` — это основной конфигурационный файл сервера Tomcat. В нем задаются параметры настройки самого сервера, такие как порты, адреса, соединения с базой данных, контейнеры и другие компоненты. Например, можно указать порты для HTTP и AJP-соединений, настройки логирования и параметры безопасности. Изменения в `server.xml` могут влиять на работу всего сервера Tomcat.
#### Как сконфигурировать Tomcat для использования другого порта, отличного от стандартного 8080?

Чтобы изменить порт в Tomcat, необходимо отредактировать файл `server.xml`, который находится в каталоге `conf` сервера. Найдите строку с портом 8080, которая выглядит как:

```
<Connector port="8080" protocol="HTTP/1.1" ... />
```
